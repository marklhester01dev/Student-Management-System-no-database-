## The Problem

Schools and small programs currently track student records manually — spreadsheets, paper forms, or ad-hoc files — with no dedicated tool for CRUD (create, read, update, delete) operations on student data.

## Why it is a problem

**No structured storage:** Spreadsheets don't enforce data integrity — duplicate Student IDs, malformed emails, or missing fields can slip in silently.

**No fast retrieval**: Manual methods make search, sort, and filter slow and error-prone once record counts grow.

**No visibility:** Without a dashboard, staff can't see live counts/breakdowns (total students, per-course, per-year-level, per-gender) without manually recalculating.

**Accessibility gap:** Manual/paper systems and generic spreadsheets aren't screen-reader friendly, which flags as an explicit concern (NVDA support).
